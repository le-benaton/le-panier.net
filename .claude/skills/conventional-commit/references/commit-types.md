# コミットタイプ 詳細ガイド

## タイプ定義と判断基準

### `feat` — 新機能

新しい機能や UI 要素を追加したとき。

**判断基準:**
- ユーザーが新しいことができるようになる
- 新しいコンポーネント・ページ・セクションを追加した
- 既存機能に新しい動作を追加した

**このプロジェクトの例:**
- `feat(): メインビジュアルにクロスフェードアニメーションを実装`
- `feat(): SelectDialog に LINE 公式アカウントリンクを追加`
- `feat(): ヘッダーにナビゲーションメニューを追加`

---

### `fix` — バグ修正

誤動作・意図しない動作を修正したとき。

**判断基準:**
- 「壊れていた」ものを直した
- エラーを修正した
- 期待通りに動かなかった挙動を修正した

**このプロジェクトの例:**
- `fix(): モバイルでメインビジュアルが正しく表示されない問題を修正`
- `fix(): SelectDialog が ?from=card なしで開いてしまう問題を修正`

---

### `style` — スタイルのみの変更

機能には影響しない、見た目だけの変更。

**判断基準:**
- CSS クラス・Tailwind クラスのみ変更
- レイアウト・色・フォント・余白の調整
- アニメーション・トランジションの視覚的調整

**このプロジェクトの例:**
- `style(): ヘッダーの背景色をホワイトに変更`
- `style(): ボタンのホバーエフェクトを追加`

---

### `refactor` — リファクタリング

動作を変えずにコード構造を改善したとき。

**判断基準:**
- 外から見た動作は同じ
- コンポーネントの分割・整理
- 重複コードの除去
- 変数名・関数名の改善

**このプロジェクトの例:**
- `refactor(): SelectDialog のリンク生成ロジックを関数に切り出し`
- `refactor(): page.tsx のセクションコンポーネントを分離`

---

### `perf` — パフォーマンス改善

パフォーマンスを改善する変更。

**判断基準:**
- 読み込み速度が改善する
- 画像・アセットを最適化した
- レンダリング効率を改善した

**このプロジェクトの例:**
- `perf(): 店舗写真を WebP 形式に変換`
- `perf(): 画像の遅延読み込みを実装`

---

### `docs` — ドキュメント

ドキュメントのみの変更。

**判断基準:**
- `.md` ファイルのみ変更
- コメントの追加・修正
- README・CLAUDE.md の更新

**このプロジェクトの例:**
- `docs(): CLAUDE.md にアーキテクチャ説明を追加`
- `docs(): コンポーネントの使用方法をコメントで説明`

---

### `chore` — その他の作業

コンテンツ更新・設定変更・依存関係更新など。

**このプロジェクトでは特に:**
- サイトのテキスト・画像コンテンツの更新（`feat` ではなく `chore`）
- `package.json`・`tsconfig.json` などの設定変更
- 依存関係のアップデート
- ファビコン・公開ファイルの追加

**このプロジェクトの例:**
- `chore(): サイトコンテンツを更新`（`ab6b753` のパターン）
- `chore(): CLAUDEのsettings.json 追加`（`61781bf` のパターン）
- `chore(): Instagramを追加`（`2a15ee0` のパターン）

---

### `test` — テスト

テストの追加・修正。

**このプロジェクトでの使用頻度:** 低（現在テストファイルなし）

---

### `build` — ビルド設定

ビルドシステム・スクリプトの変更。

**判断基準:**
- `next.config.ts` の変更
- `package.json` の scripts 変更
- `tailwind.config.ts` の変更
- Firebase Hosting 設定の変更

---

### `ci` — CI/CD

CI/CD パイプラインの変更。

**判断基準:**
- GitHub Actions ワークフローの変更
- デプロイスクリプトの変更

---

## デシジョンツリー

```
変更の種類は？
│
├─ バグを直した？
│   └─ → fix
│
├─ 新しい機能・UI を追加した？
│   └─ → feat
│
├─ ドキュメントのみ変更した？
│   └─ → docs
│
├─ CSS・スタイルのみ変更した（機能変更なし）？
│   └─ → style
│
├─ 動作を変えずにコードを整理した？
│   └─ → refactor
│
├─ パフォーマンスを改善した？
│   └─ → perf
│
├─ ビルド設定・スクリプトを変更した？
│   └─ → build
│
├─ CI/CD を変更した？
│   └─ → ci
│
└─ それ以外（コンテンツ更新・設定変更・依存関係更新など）？
    └─ → chore
```

## 破壊的変更 (`!`) の判断基準

以下のいずれかに該当する場合は `type!(): ...` を使用：

1. **URL の変更** — 既存のリンクが 404 になる
2. **クエリパラメータの動作変更** — `?from=card` の挙動が変わる
3. **コンポーネント props の変更** — 既存の使用箇所が壊れる
4. **公開 API の互換性破壊** — 外部から参照される仕様が変わる
